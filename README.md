# PDF RAG 챗봇

내 PDF 문서에 질문하면, 문서 내용을 근거로 답해주는 RAG 챗봇입니다.

## 사용 기술
- Python
- Google Gemini API (임베딩 모델 + 생성 모델)
- 코사인 유사도 기반 의미 검색

## 작동 방식

1. **PDF 읽기** — PdfReader로 PDF를 읽어 하나의 문자열로 저장합니다.
2. **청킹** — 문자열을 100글자씩 잘라 청크 리스트로 만듭니다.
3. **임베딩** — 청크들을 Gemini 임베딩 모델로 변환합니다.
   (API가 한 번에 100개까지만 처리해서, 100개씩 나눠 전송합니다.)
4. **검색** — 사용자 질문도 임베딩한 뒤, 각 청크와의 코사인 유사도를
   계산합니다. (유사도, 청크) 튜플로 저장하고 상위 3개를 고릅니다.
5. **답변 생성** — 상위 3개 청크를 맥락으로 프롬프트에 넣어,
   Gemini 생성 모델로 답변을 만듭니다.

전체 흐름: PDF → 글자 → 청킹 → 임베딩 → 검색 → 답변 생성

## 실행 방법
1. 필요한 라이브러리 설치: `pip install google-genai python-dotenv pypdf numpy`
2. `.env` 파일에 `GEMINI_API_KEY=발급받은키` 작성
3. 질문할 PDF를 `doc.pdf`라는 이름으로 폴더에 저장
   (테스트용으로는 arXiv의 'Attention Is All You Need',
    arxiv.org/abs/1706.03762 같은 논문을 추천합니다.)
4. 실행: `python rag.py`

## 배운 점
- **임베딩**: 텍스트를 의미가 담긴 숫자 벡터로 바꾸는 것. 길이는 내용과
  무관하게 모델이 정한 고정값(3072)이고, 의미는 숫자 값에 반영된다.
- **코사인 유사도**: 두 벡터의 길이가 아니라 방향(각도)으로 의미의
  유사성을 잰다. 1에 가까울수록 비슷하고, numpy로 계산했다.
- **배치 처리**: API가 한 번에 100개까지만 받아서, 청크를 100개씩 나눠
  전송했다. 결과를 줄이는 게 아니라 전송을 쪼개는 것.
- **환각 방지**: 프롬프트에 "문서에 없으면 모른다고 답하라"는 지시와
  관련 맥락을 함께 넣어, 모델이 지어내지 않게 했다.
- **환경변수 관리**: API 키를 코드에 직접 쓰지 않고 .env에 분리해
  load_dotenv로 불러왔다. .gitignore로 키 노출도 방지했다.
